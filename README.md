# 🌙 DreamDrift — سُكون

رفيق ليلي لتتبّع النوم والمزاج، مع تمارين تنفّس، مازج أصوات هادئة، واسترخاء موجّه بالصوت. تطبيق ويب تقدّمي (PWA) بملف واحد — بلا خادم خلفي، بلا بناء (build step)، كل البيانات تبقى على جهاز المستخدم.

**الإصدار الحالي:** 1.3.0

## ✨ المزايا

- تتبّع المزاج ونمط النوم اليومي
- تمارين تنفّس موجّهة (٤-٤-٤) بصوت مرافق
- مازج أصوات (مطر، أمواج، غابة، نار، رياح، ضوضاء بنية) مبني بالكامل بـ Web Audio API — بلا ملفات صوتية خارجية
- استرخاء موجّه بالصوت (مسح جسدي) عبر `speechSynthesis`، خلف بوابة جودة تتحقق من توفّر صوت مناسب
- **التحليل التفصيلي**: أربعة "وكلاء" تحليليون يقرؤون سجلات النوم — المِصفاة (تدقيق البيانات) ← البوصلة (تشخيص النمط) ← الفنار (فرز السلامة) ← المرساة (خطوة أسبوعية واحدة)
- إشعارات محلية لتذكير الاسترخاء ومنبّه الاستيقاظ عبر Service Worker (بلا خادم push)
- عمل كامل دون إنترنت بعد أول زيارة
- عربي/إنجليزي، وضع داكن/فاتح
🔔 كيف تعمل الإشعارات

المتصفح لا يسمح بمنبّهات حقيقية بلا خادم Push، لذا يعتمد `sw.js` على نمط شائع لتطبيقات PWA الشخصية:

- عند فتح التطبيق، يُرسِل الصفحة رسالة `CHECK_NOTIFICATIONS_NOW` لـ Service Worker.
- إن دعم المتصفح `periodicSync` ومُنح الإذن، يُفحص الجدول كل ١٥ دقيقة تلقائياً.
- الفحص يقارن الوقت الحالي بالموعد المحفوظ (نافذة ٥ دقائق) عبر IndexedDB، ولا يُطلق الإشعار أكثر من مرة في اليوم.
- **القيد:** إن بقي التطبيق مغلقاً لفترة طويلة على متصفح لا يدعم `periodicSync` (أغلب متصفحات iOS مثلاً)، فالإشعار يُطلَق عند الفتح التالي إن كان ضمن النافذة، لا بالضرورة في وقته تماماً. هذا قيد منصات لا قيد تصميم.

## 🧪 التطوير محلياً

ملف واحد، لكن لتفعيل Service Worker بشكل صحيح يجب تقديمه عبر HTTP (لا `file://`):

```bash
python3 -m http.server 8080
# ثم افتح http://localhost:8080
```

## 📄 بنية البيانات

تُحفظ كل بيانات المستخدم محلياً عبر `localStorage` بنظام إصدارات (`SCHEMA_VERSION`) مع ترحيل تلقائي عند تغيّر البنية بين التحديثات. الإصدار الحالي: `4`.
